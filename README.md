# Git & GitHub para CCNA Automation 🚀

Repositório de estudo para quem está se preparando para o **CCNA Automation** e precisa dominar **Git e GitHub** — ferramentas essenciais no dia a dia de automação de redes (versionar playbooks, configs, scripts Python, YAML, etc).

---

## 📖 O que tem aqui

- **[tutorial-git-github.md](./tutorial-git-github.md)** — Tutorial completo do zero: o que é Git, configuração, comandos básicos, GitHub e branches. Comece por aqui se você nunca usou Git. **Inicie aqui!**

- **Labs práticos** — pastas `lab1` a `lab6`, uma para cada assunto do tutorial, com arquivos de exemplo prontos (incluindo configs de rede e YAML, no espírito do CCNA Automation). **Deep Dive!**

- **`lab-final`** — desafio que junta tudo: você vai inicializar, versionar, subir pro GitHub e trabalhar com branches em um cenário único, simulando um projeto real de automação. **Desafio!!**

## ▶️ Como começar

### 1. Clone este repositório

```bash
git clone https://github.com/SEU-USUARIO/NOME-DO-REPO.git
cd NOME-DO-REPO
```

Pronto — você já tem o tutorial e todos os labs na sua máquina.

### 2. Leia o tutorial

Abra `tutorial-git-github.md` e siga os conceitos básicos.

### 3. Faça os labs em ordem

Entre em cada pasta (`lab1`, `lab2`...) e siga as instruções do `INSTRUCOES.md` de cada uma.

### 4. Finalize com o Lab Final

A pasta `lab-final` simula um cenário de automação de rede completo, unindo tudo que você praticou.

---

## 🎯 Por que isso importa para o CCNA Automation?

No exame e na prática de automação de redes, você vai:

- Versionar **playbooks Ansible**, scripts **Python** e arquivos **YAML/JSON** de inventário
- Colaborar em equipe sem sobrescrever configurações
- Usar branches para testar mudanças de automação sem afetar produção
- Subir e baixar projetos do GitHub como parte do fluxo de trabalho com **NetDevOps**

Dominar Git não é opcional — é parte da base do **CCNA Automation**.
